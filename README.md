# Terrain Nav

Прототип навигационной привязки БПЛА при потере ГНСС. Он сопоставляет профиль высоты земли, полученный из барометра и радиовысотомера, с цифровой моделью рельефа (DEM), поддерживает несколько гипотез движения и оценивает текущие координаты, азимут, путевую скорость и уверенность.

Комплекс выдает навигационную оценку, но **не является автопилотом** и не формирует управляющие команды.

## Принцип работы

Для каждого момента времени вычисляется оценка высоты земли:

```text
H_ground(t) = H_baro(t) - H_radio(t)
```

Одна высота встречается во множестве точек карты, поэтому координату по одному измерению определить нельзя. Алгоритм накапливает временное окно и ищет путь, вдоль которого профиль DEM совпадает с измеренным по абсолютным высотам и по форме: нормализованным высотам, уклонам, рангам и RMSE после удаления постоянного вертикального смещения.

**Необходимо реализовать непрерывную функцию выборки высоты DEM `z = f(x, y)`, основанную на локальной интерполяции дискретной сетки рельефа.** В этом проекте такую функцию предоставляет `DEMInterpolator`. Основной режим — bilinear: он использует четыре соседние ячейки, быстро векторизуется и не создает новых экстремумов. Опциональный bicubic предназначен только для offline-анализа и требует SciPy.

## Поиск движения

Кандидат описывается азимутом и физической путевой скоростью. Для временных отсчетов `t` строится путь:

```text
s(t) = ground_speed_mps * (t - t0)
x(t) = x_start + s(t) * sin(azimuth)
y(t) = y_start + s(t) * cos(azimuth)
```

На первом окне скорость перебирается в пределах `0.7 * v0 … 1.3 * v0`. Если известен начальный азимут, поиск идет в секторе ±45°; иначе — по всем 360°. После потери ГНСС скорость не считается известной: каждое следующее окно перебирает скорость около предыдущей гипотезы и азимут в пределах допустимого поворота.

Окна формируются по timestamp каждого NMEA-измерения, а не по заданной или вычисленной частоте. Поэтому поддерживается нерегулярный поток примерно 1–10 Гц. При стандартном окне 10 с в score попадают все измерения за полные 10 с: например, 31 точка при 3 Гц. Перекрывающаяся часть не теряется, но движение продолжается только по новым timestamps. Каждые несколько окон расширенный пул гипотез повторно оценивается по всей истории от первого измерения до текущего момента.

Вместо необратимого выбора одной точки используется beam search. После окна сохраняются top-K гипотез с собственными координатами, скоростью, курсом, score и историей пути. При низкой уверенности beam, диапазоны скорости/азимута и фактическая история поиска расширяются.

Быстрый score объединяет Pearson нормализованных высот, Pearson уклонов, Spearman и bias-corrected RMSE. NMI и DTW применяются только для re-ranking лучших кандидатов. Дополнительно действуют штрафы за скачок скорости, резкий поворот, выход за карту и малоизменчивый рельеф.

Confidence зависит от score, разрыва между первой и второй гипотезой, стабильности движения и информативности рельефа. Информативность учитывает разброс высот, разброс уклонов и локальные экстремумы. При стандартном отклонении профиля менее 5 м confidence принудительно остается низким.

## Установка

Требуется Python 3.13+.

```bash
python3.13 -m venv .venv
.venv/bin/pip install -e '.[test]'
```

Минимальный контур использует NumPy, Matplotlib и PyYAML. GeoTIFF и bicubic доступны через дополнительные зависимости:

```bash
.venv/bin/pip install -e '.[offline]'
```

Форматы DEM: `.npy`, `.npz`, а при наличии rasterio — `.tif/.tiff`. Массив имеет вид `dem[y, x]`, координаты внутри алгоритма локальные метрические: X — восток, Y — север.

## Быстрый запуск

```bash
.venv/bin/python -m terrain_nav generate-test \
  --terrain mixed \
  --output-dir data/demo \
  --baro-altitude 1500 \
  --sample-rate 10 \
  --duration 180 \
  --start-x 1000 \
  --start-y 1500 \
  --initial-speed 30 \
  --azimuth 80 \
  --radio-noise-std 0.8 \
  --radio-outlier-probability 0.002 \
  --motion turn
```

```bash
.venv/bin/python -m terrain_nav localize \
  --dem data/demo/dem.npy \
  --config data/demo/config.yaml \
  --nmea data/demo/radio_altimeter.nmea \
  --start-x 1000 \
  --start-y 1500 \
  --initial-speed 30 \
  --initial-azimuth 80 \
  --baro-altitude 1500 \
  --output-dir results/demo \
  --preview-trajectory
```

Флаг `--preview-trajectory` открывает интерактивное окно Matplotlib до сохранения `trajectory.png`. Можно масштабировать и перемещать карту стандартной панелью Matplotlib; после закрытия окна PNG сохраняется, а CLI продолжает работу. Флаг необязателен и требует запуска в графической сессии.

Без `--initial-azimuth` первое окно проверяет все направления. Парсер NMEA устойчив к пустым/поврежденным строкам; `--verify-checksum` включает отбрасывание строк с неверной контрольной суммой.

Полный справочник по аргументам обеих команд находится в [CLI_FLAGS.md](CLI_FLAGS.md). Для быстрого запуска также доступен `Makefile`:

```bash
make generate-test MOTION=sharp-mixed START_X=1000 START_Y=1500
make localize START_X=1000 START_Y=1500
```

Результаты:

- `result.json` — лучшая оценка, альтернативы и компоненты score;
- `trajectory.png` — DEM, лучшая и альтернативные траектории;
- `score_heatmap.png` — азимут × путевая скорость для первого окна;
- `profile_comparison.png` — исходные, нормализованные профили и уклоны.

## Тесты

```bash
.venv/bin/pytest
```

Синтетический генератор поддерживает гористый, холмистый, плоский, долинный и смешанный рельеф, прямой полет, плавные повороты/изменения скорости и отдельные ступенчатые режимы `sharp-turn`, `sharp-speed-change`, `sharp-mixed`. `true_path.npz` используется только для проверки и отображения, но не участвует в локализации.

## Ограничения

- плоский и повторяющийся рельеф плохо различает гипотезы;
- результат зависит от разрешения и вертикальной точности DEM;
- смещение барометра частично компенсируется, но переменный дрейф ухудшает результат;
- шум, выбросы и пропуски радиовысотомера требуют накопления профиля;
- модель окна предполагает почти постоянные скорость и азимут внутри короткого сегмента;
- прототип не заменяет инерциальную систему, GNSS или сертифицированный автопилот.
