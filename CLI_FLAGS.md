# Флаги командной строки Terrain Nav

Команды можно запускать как модуль Python:

```bash
python -m terrain_nav generate-test ...
python -m terrain_nav localize ...
```

При работе из репозитория вместо `python` обычно используется `.venv/bin/python`.

## `terrain_nav generate-test`

Создает синтетическую DEM, истинную траекторию, показания радиовысотомера в NMEA и конфигурацию поиска. Команда не запускает локализацию.

| Флаг | Обязательный | По умолчанию | Описание |
|---|---:|---:|---|
| `-h`, `--help` | нет | — | Показать встроенную справку. |
| `--terrain` | нет | `mixed` | Тип DEM: `mixed`, `mountainous`, `hilly`, `flat` или `valley`. |
| `--motion` | нет | `straight` | Модель движения, описанная ниже. |
| `--output-dir` | да | — | Каталог для `dem.npy`, `radio_altimeter.nmea`, `true_path.npz` и `config.yaml`. |
| `--baro-altitude` | нет | `1500.0` | Постоянная барометрическая высота над уровнем моря, м. |
| `--sample-rate` | нет | `10.0` | Частота моделирования радиовысотомера, Гц. |
| `--duration` | нет | `60.0` | Продолжительность полета, с. |
| `--start-x` | нет | `1000.0` | Стартовая координата X — восток, м. |
| `--start-y` | нет | `1000.0` | Стартовая координата Y — север, м. |
| `--initial-speed` | нет | `50.0` | Начальная путевая скорость, м/с. |
| `--azimuth` | нет | `73.0` | Начальный азимут от севера по часовой стрелке, градусы. |
| `--resolution` | нет | `30.0` | Размер ячейки синтетической DEM, м. |
| `--radio-noise-std` | нет | `0.8` | Стандартное отклонение гауссова шума радиовысотомера, м. Значение `0` отключает обычный шум. |
| `--radio-outlier-probability` | нет | `0.002` | Вероятность отдельного импульсного выброса для каждого измерения, диапазон `0…1`. Значение `0` отключает выбросы. |
| `--seed` | нет | `42` | Seed генератора случайных чисел для воспроизводимого шума. |

### Режимы `--motion`

| Значение | Поведение |
|---|---|
| `straight` | Постоянные азимут и скорость. |
| `turn` | Азимут плавно увеличивается на 24° за весь полет. |
| `yaw` | Азимут колеблется на ±4° с периодом 12 с. |
| `speed-change` | Скорость плавно увеличивается до 118% начальной. |
| `mixed` | Плавное изменение азимута и скорости одновременно. |
| `sharp-turn` | Менее плавный курс: два мгновенных изменения азимута по +15° на 1/3 и 2/3 полета. |
| `sharp-speed-change` | Резкие ступени скорости: 100% → 130% на 35% полета → 80% на 70% полета. |
| `sharp-mixed` | Одновременные ступенчатые изменения азимута и скорости из двух предыдущих режимов. |

Пример:

```bash
python -m terrain_nav generate-test \
  --terrain mountainous \
  --motion sharp-mixed \
  --output-dir data/sharp_demo \
  --duration 60 \
  --start-x 1400 \
  --start-y 1200 \
  --initial-speed 45 \
  --azimuth 65 \
  --radio-noise-std 2.5 \
  --radio-outlier-probability 0.01
```

Старт и вся траектория должны находиться внутри DEM. При стандартной карте `256 × 256` и разрешении 30 м допустимый диапазон узлов составляет `0…7650` м по обеим осям. Генератор завершится ошибкой, если траектория выйдет за карту.

## `terrain_nav localize`

Загружает DEM и NMEA, строит профиль земли, выполняет beam search и сохраняет результат с графиками.

| Флаг | Обязательный | По умолчанию | Описание |
|---|---:|---:|---|
| `-h`, `--help` | нет | — | Показать встроенную справку. |
| `--dem` | да | — | Путь к DEM в формате NPY, NPZ или GeoTIFF при наличии rasterio. |
| `--config` | нет | встроенные настройки | YAML с разрешением/началом DEM и параметрами поиска. |
| `--nmea` | да | — | Файл показаний радиовысотомера NMEA-0183. |
| `--start-x` | да | — | Последняя достоверная координата X, м. Для синтетики должна совпадать с `generate-test --start-x`. |
| `--start-y` | да | — | Последняя достоверная координата Y, м. Для синтетики должна совпадать с `generate-test --start-y`. |
| `--initial-speed` | да | — | Скорость в момент потери ГНСС, м/с. |
| `--initial-azimuth` | нет | полный поиск 0…359° | Азимут в момент потери ГНСС. Если указан, первый поиск ограничивается сектором ±45°. |
| `--baro-altitude` | да | — | Барометрическая высота над уровнем моря, м. |
| `--output-dir` | да | — | Каталог для `result.json` и PNG-графиков. |
| `--verify-checksum` | нет | выключено | Отбрасывать строки NMEA с неправильной контрольной суммой. |
| `--preview-trajectory` | нет | выключено | Показать интерактивное окно траектории до сохранения `trajectory.png`. После закрытия окна выполнение продолжается. |

Временные окна локализации строятся непосредственно по timestamps из NMEA. Частота не задаётся в `localize` и может изменяться в диапазоне примерно 1–10 Гц. По умолчанию используются полные 10-секундные окна с 50% перекрытием; периодически top-K пересчитывается по всему накопленному профилю.

Пример:

```bash
python -m terrain_nav localize \
  --dem data/sharp_demo/dem.npy \
  --config data/sharp_demo/config.yaml \
  --nmea data/sharp_demo/radio_altimeter.nmea \
  --start-x 1400 \
  --start-y 1200 \
  --initial-speed 45 \
  --initial-azimuth 65 \
  --baro-altitude 1500 \
  --output-dir results/sharp_demo \
  --preview-trajectory
```

Команда сохраняет:

- `result.json`;
- `trajectory.png` — top-K, лучшая и, для синтетического теста, истинная траектория;
- `score_heatmap.png`;
- `profile_comparison.png`.

## Быстрый запуск через Makefile

```bash
make generate-test
make localize
make localize-preview
make demo
```

Значения можно переопределять без редактирования Makefile:

```bash
make demo \
  TERRAIN=mixed \
  MOTION=sharp-mixed \
  START_X=1400 \
  START_Y=1200 \
  INITIAL_SPEED=45 \
  AZIMUTH=65
```

Основные переменные Makefile совпадают по смыслу с флагами CLI: `DATA_DIR`, `RESULTS_DIR`, `TERRAIN`, `MOTION`, `BARO_ALTITUDE`, `SAMPLE_RATE`, `DURATION`, `START_X`, `START_Y`, `INITIAL_SPEED`, `AZIMUTH`, `RESOLUTION`, `RADIO_NOISE_STD`, `RADIO_OUTLIER_PROBABILITY` и `SEED`.

Для полного поиска начального азимута передайте пустое значение:

```bash
make localize INITIAL_AZIMUTH=
```

Для проверки checksum:

```bash
make localize VERIFY_CHECKSUM=1
```
